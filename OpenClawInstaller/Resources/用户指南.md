# GetClawHub 用户指南

> 适用版本：1.1.32+ | 系统要求：macOS 13.0 及以上

---

## 快速开始

### 首次安装

打开 GetClawHub 后，安装向导会引导你完成 6 步配置：

1. 点击 **开始安装** 开始
2. 等待环境检测完成（自动检查系统版本、磁盘空间等）
3. 安装 Node.js 运行时（优先使用应用内置的 Node.js 24，无需联网）
4. 安装 OpenClaw 核心服务（自动完成）
5. 设置网关 — 填写 **端口**（默认 18789 即可）和 **Auth Token**（用于保护接口安全，自行设定一个密码）
6. 点击 **完成**，进入主界面

> 首次启动时窗口会自动最大化，方便查看完整界面。

### 已安装用户

如果 OpenClaw 已安装，启动后会显示 **Open Dashboard** 按钮，直接进入主界面。如需重新安装，可点击 **Uninstall** 后重新开始。

### 登录

部分版本需要登录才能使用完整功能：
1. 应用启动后会弹出登录提示
2. 点击 **Log In**，浏览器自动打开登录页面
3. 在浏览器中完成登录后，应用自动识别登录状态

登录后可使用 GetClawHub 官方 AI 服务、查看账单、管理 API Key 等功能。

- **Persona** — 设定 AI 的性格和身份

- **Configuration** — 配置网关和模型供应商
- **Skills** — 管理 AI 技能

- **Channels** — 连接 Telegram、Discord、微信等平台
- **Plugins** — 启用/禁用插件
- **Cron** — 设置定时自动任���

侧边栏底部：
- 点击 **太阳/月亮图标** 可切换浅色/深色主题
- 点击 **版本号** 可检查更新
- 点击 **?** 图标打开帮助助手窗口

侧边栏顶部 **地球图标** 可切换界面语言（支持中文、英文、日文等 26 种语言）。

### 菜单栏图标

应用运行时，macOS 顶部菜单栏会显示状态图标：
- 🟢 服务正在运行
- 🔴 服务已停止
- 🟡 服务启动中

点击图标可快速启动/停止服务、打开面板或检查更新。

## Chat — 和 AI 对话

这是你最常用的页面，用来和 AI 助手交流。

### 发送消息

1. 点击底部输入框，输入你想说的内容
2. 按 **回车键** 发送（按 **Shift + 回车** 可以换行）
3. 等待 AI 回复（会显示 "Thinking..." 动画）

### 附加文件

输入框左侧有 **回形针按钮（📎）**，点击可选择文件附加到消息中。支持的文件类型包括：图片、PDF、纯文本、音频、视频、Office 文档（Word、Excel、PPT）、CSV、JSON、Markdown 等。

你也可以直接将文件**拖拽**到输入区域。已添加的文件会显示为缩略图预览，点击 **✕** 可移除。支持一次附加多个文件。

### 切换 AI 助手

如果你创建了多个 AI（子代理），有两种方式切换：

**方式一**：输入框上方的下拉菜单选择。

**方式二**：在输入框中输入 `@`，会弹出 Agent 列表。继续打字可筛选，用 **↑↓ 键** 选择，按 **回车** 或 **Tab** 确认。选中后直接在后面输入消息内容即可发送给指定 Agent。

不同 AI 会用不同的头像标识回复。

### 取消 AI 回复

AI 思考中时，点击思考动画旁的 **取消按钮** 可中止当前回复。

### 清空对话

点击输入框上方的 **New** 按钮，清空当前对话，开始新会话。

### 使用斜杠命令

在输入框中输入 `/`，会弹出命令列表。你可以：

- 继续打字筛选命令（如输入 `/he` 会只显示 `/help`）
- 用 **↑↓ 键** 上下移动选择
- 按 **回车** 或 **Tab** 确认
- 按 **Esc** 关闭列表

**常用命令：**

| 输入 | 作用 |
|------|------|
| `/help` | 查看帮助信息 |
| `/new` | 重置对话（清空聊天记录） |
| `/status` | 查看当前会话状态 |
| `/agents` | 查看所有可用 AI |
| `/agent 名字` | 切换到指定 AI |
| `/models` | 查看可用模型列表 |
| `/model 名字` | 切换到指定模型 |
| `/think high` | 开启深度思考模式 |
| `/skills` | 浏览并选择一个技能使用 |
| `/skills 名称` | 直接使用指定技能 |
| `/abort` | 中止当前正在运行的任务 |
| `/exit` | 退出应用 |

> 不带参数的命令选中后直接执行；需要参数的命令（如 `/agent`）选中后会自动填入输入框，等你补充参数后再按回车发送。

### 查看历史消息

输入框**为空**时，按 **↑** 键可以逐条回看之前发过的消息，按 **↓** 键反向翻阅。历史记录最多保留 20 条，关闭应用后依然保留。

### 长时间任务与后台运行

当 AI 任务运行超过 **60 秒**时，输入框旁会出现 **转后台** 按钮。点击后：

- 任务继续在后台运行，输入框解锁，你可以继续和 AI 对话或切换到其他 Agent
- 消息气泡上会显示 "后台运行中..." 状态指示
- 任务完成后，聊天底部会出现 ✅ 完成通知，点击 **查看结果 ↑** 可跳转到对应的回复内容

> 如果超过 **120 秒** 仍未完成，任务会自动转入后台。

**超时机制：** 任务最长运行 **30 分钟**。如果超时，进程会被终止，底部显示超时提示。你可以选择重新发送或切换到其他 Agent。

## Persona — 设定 AI 性格

在这里定义你的主 AI 助手是谁、性格如何。

页面顶部的**身份卡片**会实时展示 AI 的头像 emoji、名称和风格。

### 四个可编辑文件

点击每个区域的标题展开编辑：

| 文件 | 用来做什么 | 举例 |
|------|------------|------|
| **IDENTITY** | 定义 AI 的名字、头像、物种 | "我叫小助，是一只数码猫" |
| **SOUL** | 定义性格和行为风格 | "友善、幽默、回答简洁" |
| **USER** | 告诉 AI 你是谁、你的偏好 | "我是开发者，喜欢用 Python" |
| **MEMORY** | AI 的长期记忆 | "用户的项目用的是 React" |

### 如何编辑

1. 点击标题左边的 **箭头** 展开区域
2. 点击 **铅笔图标** 进入编辑模式，直接修改文本
3. 点击 **眼睛图标** 可预览渲染后的效果
4. 修改后会出现 **橙色圆点** 提示未保存
5. 点击 **Save** 按钮保存

## Multi-Agent — 管理多个 AI

你可以创建多个不同性格、不同用途的 AI 子代理。

### 创建新 AI

1. 点击页面顶部 **New Agent**
2. 填写 **Agent ID**（英文小写 + 数字 + 连字符，如 `code-helper`）
3. 可选填 **Display Name**（显示名称）
4. 可选择专用模型（默认继承全局模型）
5. 点击创建

### 编辑子代理

点击子代理卡片展开，可以：
- 编辑它的 IDENTITY、SOUL、MEMORY 文件（操作方式和 Persona 页面一样）
- 修改绑定的模型
- 查看工作目录

### 使用子代理对话

创建完成后，回到 **Chat** 页面，在输入框上方的下拉菜单中选择你创建的 AI 即可切换对话。也可以用 `@` 快速切换。

### 删除子代理

点击卡片右上角的 **垃圾桶图标**，确认后删除。默认 AI 不可删除。

## Configuration — 配置

配置页面分为两个区域：**GetClawHub 官方服务**（蓝色边框）和**自定义 API 供应商**。通过左侧的圆形单选按钮选择当前使用的服务来源。

### GetClawHub 官方服务（推荐）

> 需要登录后使用。

选中此选项后：
- 显示当前会员等级和预算信息
- API Key 自动从账号同步，也可手动输入
- 模型列表根据会员等级自动显示（不可编辑）
- 点击 **Manage** 可跳转到官网管理账号

### 自定义 API 供应商

选中此选项后，可以手动配置第三方 AI 服务：

1. 在 **Provider** 下拉框选择供应商（如阿里云百炼、OpenAI、DeepSeek 等）
2. **API Base URL** 会自动填充（也可手动修改）
3. 填入你的 **API Key**
4. 模型列表从预设加载，可以添加或删除模型

> 切换供应商时会弹出确认框，确认后 API Key 会被清空，模型列表替换为新供应商的预设。

### 修改网关端口和令牌

- **Port**：默认 18789，一般不需要修改
- **Auth Token**：点击眼睛图标可查看/隐藏，用于保护网关安全

### 保存配置

| 按钮 | 用途 |
|------|------|
| **Save** | 保存修改 |
| **Save & Restart** | 保存并重启服务（修改端口、供应商后建议用这个） |
| **Reset** | 放弃所有未保存的修改 |

> 保存时会自动更新所有 Agent 的模型引用，确保切换供应商后不会出现模型失效。

## Channels — 连接聊天平台

让你的 AI 接入 Telegram、Discord、Slack、微信、钉钉、飞书等平台，随时随地和 AI 对话。

1. 点击 **Add Channel**

- **绿色** — 已连接，正常工作

## Cron — 定时自动任务

让 AI 按照设定的时间自动执行任务，比如每天早上 9 点推送天气、每周一生成报告。

### 创建定时任务

1. 点击 **Add Job**
2. 填写任务名称（如 `morning-weather`）
3. 设置执行时间（Cron 表达式）：

| 表达式 | 含义 |
|--------|------|
| `0 9 * * *` | 每天早上 9:00 |
| `0 9 * * 1` | 每周一早上 9:00 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 9,18 * * *` | 每天 9:00 和 18:00 |

4. 选择时区（默认 Asia/Shanghai）
5. 选择执行的 AI（如果有多个子代理的话）
6. 选择会话模式：
   - **Isolated** — 每次新建独立会话（推荐，互不干扰）
   - **Main** — 在主会话中执行（可以延续上下文）
7. 填写要执行的指令（如"查询今天的天气并推送到 Telegram"）
8. 点击创建

### 管理任务

- 点击 **播放/暂停按钮** 启用或暂停任务
- 点击 **Run Now** 立即执行一次（不等定时触发）
- 点击 **垃圾桶图标** 删除任务

## Logs — 查看日志

服务运行日志用于排查问题。

### 基本操作

| 操作 | 方法 |
|------|------|
| 搜索日志 | 在搜索栏输入关键字（如 "error"） |
| 自动刷新 | 打开 **Auto** 开关，每 3 秒自动更新 |
| 手动刷新 | 点击 **刷新按钮** |
| 导出日志 | 点击 **Export**，保存为 .txt 文件 |
| 打开日志文件 | 点击 **Open File**，在 Finder 中定位文件 |

### 日志颜色含义

- **红色** — 错误（ERROR），需要关注
- **橙色** — 警告（WARN），可能有问题
- **蓝色** — 信息（INFO），正常运行记录

## 常见问题

### 服务启动不了怎么办？

1. 进入 **Status** 页面，查看错误信息
2. 进入 **Logs** 页面，搜索 "error" 查找报错原因
3. 点击 **Doctor** 生成诊断报告
4. 尝试点击 **Restart** 重启服务

### AI 不回复怎么办？

1. 检查 **Status** 页面，确认服务是否正在运行
2. 检查 **Configuration** 页面，确认 API Key 是否正确
3. 查看 **Logs** 是否有请求错误
4. 尝试在 Chat 中输入 `/new` 重置会话

### 提示 "API Key may not exist"？

你的 API Key 可能已被删除或过期。请访问 [API Key 管理页面](https://www.getclawhub.com/member/api-keys/) 检查 Key 状态，必要时重新创建。

### 提示 "API Key has exceeded its budget"？

你的 API Key 已用完当月预算。请访问 [账单页面](https://www.getclawhub.com/member/billing/) 查看用量详情，或等待下月预算重置。

### 如何切换模型？

**方法一**：进入 **Models** 页面，找到想用的模型，点击 **Set Default**

**方法二**：在 Chat 中输入 `/model 模型名称`（如 `/model deepseek-v4-pro`）

### 如何切换供应商？

进入 **Configuration** 页面，选择另一个供应商的单选按钮，确认切换后填入 API Key，点击 **Save & Restart**。

### 如何更新应用？

- 应用启动时自动检查更新
- 侧边栏底部如果显示绿色 **Update**，点击即可下载安装
- 也可以手动检查：点击版本号旁的更新按钮

### 每次打开都显示安装向导？

如果 OpenClaw 已安装但仍显示安装页面，可能是 Node.js 或 OpenClaw 的路径未被正确检测。请确认终端中 `openclaw --version` 能正常运行，如有问题请通过安装向导重新安装。
