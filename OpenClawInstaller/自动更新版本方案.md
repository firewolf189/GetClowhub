Sparkle 自动更新方案                                                    
                                                                          
  整体流程                                                                
                                                                          
  发版时:                                                                 
  build_dmg.sh 构建 DMG → sign_update 签名 → 生成 appcast.xml → 上传      
  GitHub Release                                                          
                                                                          
  用户端:                                                                 
  App 启动 → Sparkle 拉取 appcast.xml → 发现新版本 → 弹窗提示             
  → 用户点"安装更新" → 后台下载 DMG → 自动替换 .app → 重启                
                                                                          
  ---                                                                     
  需要做的事情                                                            
                                                                          
  1. 集成 Sparkle 框架                                                    
                                                                          
  通过 SPM 添加 Sparkle 依赖（https://github.com/sparkle-project/Sparkle）
  ，项目当前零外部依赖，这将是第一个。                                    
                                                                          
  2. 生成 EdDSA 密钥对                                                    
                                                                          
  用 Sparkle 自带的 generate_keys 工具生成，只需执行一次：                
  - 私钥自动存入 macOS 钥匙串（构建签名时用）                             
  - 公钥写入 Info.plist 的 SUPublicEDKey 字段                             
                                                                          
  3. 配置 Info.plist                                                      
                                                                          
  新增以下配置项：                                                        
  ┌──────────────────────────┬────────────────────┬─────────────────────┐ 
  │           Key            │         值         │        说明         │ 
  ├──────────────────────────┼────────────────────┼─────────────────────┤ 
  │ SUPublicEDKey            │ 生成的公钥         │ EdDSA 验证公钥      │ 
  ├──────────────────────────┼────────────────────┼─────────────────────┤ 
  │ SUFeedURL                │ appcast.xml 的 URL │ 更新源地址          │ 
  ├──────────────────────────┼────────────────────┼─────────────────────┤ 
  │ SUEnableAutomaticChecks  │ YES                │ 自动检查更新        │ 
  ├──────────────────────────┼────────────────────┼─────────────────────┤ 
  │ SUScheduledCheckInterval │ 86400              │ 检查间隔（秒），1天 │ 
  └──────────────────────────┴────────────────────┴─────────────────────┘ 
  4. 改造代码                                                             
  改动: 删除 UpdateChecker.swift                                          
  说明: 整个文件替换为 Sparkle，现有代码未接入任何地方，无影响            
  ────────────────────────────────────────                                
  改动: 新建 SparkleUpdater.swift                                         
  说明: 封装 SPUStandardUpdaterController，约 20 行代码                   
  ────────────────────────────────────────                                
  改动: 修改 AppDelegate.swift                                            
  说明: 菜单中添加 "Check for Updates" 项，绑定 Sparkle                   
  ────────────────────────────────────────                                
  改动: 修改 OpenClawInstallerApp.swift                                   
  说明: 初始化 Sparkle updater                                            
  Sparkle 自带完整的更新                                                  
  UI（检查中、发现新版本、下载进度、安装确认），不需要自己写界面。        
                                                                          
  5. 改造 build_dmg.sh                                                    
                                                                          
  在构建流程末尾增加：                                                    
                                                                          
  构建 app → 签名 → 打包 DMG → EdDSA 签名 DMG → 生成/更新 appcast.xml     
                                                                          
  需要加的步骤：                                                          
  - 用 sign_update 对 DMG 生成 EdDSA 签名                                 
  - 生成 appcast.xml（包含版本号、下载地址、签名、文件大小）              
                                                                          
  6. 版本号管理                                                           
                                                                          
  统一版本号来源，解决当前的版本分散问题：                                
  ┌─────────────────────┬──────────────────────────────────────┐          
  │        现状         │                 改为                 │          
  ├─────────────────────┼──────────────────────────────────────┤          
  │ Info.plist 写死 1.0 │ 保持作为唯一来源                     │          
  ├─────────────────────┼──────────────────────────────────────┤          
  │ 代码硬编码 "1.0.0"  │ 从 Bundle.main 读取                  │          
  ├─────────────────────┼──────────────────────────────────────┤          
  │ 构建时不自增        │ build_dmg.sh 中可选自增 build number │          
  └─────────────────────┴──────────────────────────────────────┘          
  7. appcast.xml 托管                                                     
                                                                          
  appcast.xml 是一个 RSS 格式的 XML 文件，告诉 Sparkle                    
  最新版本信息。托管方式：                                                
  方案: GitHub Pages（推荐）                                              
  说明: 仓库开启 Pages，appcast.xml 放在 docs/ 或 gh-pages 分支           
  ────────────────────────────────────────                                
  方案: GitHub Raw                                                        
  说明: 直接放仓库里，用 raw.githubusercontent.com 地址                   
  ────────────────────────────────────────                                
  方案: 自有服务器                                                        
  说明: 放在任意可公开访问的 HTTP 地址                                    
  DMG 文件通过 GitHub Releases 的 Assets 托管（天然 CDN，不占仓库体积）。 
                                                                          
  appcast.xml 示例结构：                                                  
                                                                          
  <rss                                                                    
  xmlns:sparkle="http://www.andymatuschak.org/xml-namespaces/sparkle">    
    <channel>                                                             
      <title>OpenClaw Helper Updates</title>                              
      <item>                                                              
        <title>Version 1.1.0</title>                                      
        <sparkle:version>1.1.0</sparkle:version>                          
        <sparkle:shortVersionString>1.1.0</sparkle:shortVersionString>    
        <description>更新内容...</description>                            
        <enclosure url="https://github.com/.../OpenClawHelper.dmg"        
                   length="209715200"                                     
                   type="application/octet-stream"                        
                   sparkle:edSignature="签名字符串" />                    
      </item>                                                             
    </channel>                                                            
  </rss>                                                                  
                                                                          
  ---                                                                     
  发版工作流（改造后）                                                    
                                                                          
  每次发新版本：                                                          
                                                                          
  1. 修改 Xcode 中的 MARKETING_VERSION（如 1.0.0 → 1.1.0）                
  2. 运行 build_dmg.sh → 自动构建、签名、生成 appcast.xml                 
  3. 上传 DMG 到 GitHub Release                                           
  4. 提交更新后的 appcast.xml 到仓库                                      
                                                                          
  可以进一步用 GitHub Actions 自动化，但初期手动即可。
